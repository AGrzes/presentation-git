Git

Distributed revision control system with an emphasis on speed, data integrity, and support for distributed, non-linear workflows.

Background

  • Created by Linus Torvalds and linux kernel developers in 2005
  • Maintained by community
  • MIT license

Advantages

Content

  • Distributed
  • Fast
  • Open Source
  • Easy branching and merging
  • Ensures content and history integrity
  • Popular

Distributed

  • No need for central server
  • Everyone have a full backup copy of history
  • Easy offline work
  • Various workflows possible

Usage

  • Local version tracking
  • Team collaboration
  • Advanced propagation workflows
  • Automation
  • Cloud deployment

Caveats

  • Binary file handling
  • Coarse grained security
  • No locking

Basic Terms

  • Repository - a structure tracking version history
  • Work Tree - local directory when work is performed
  • Index / Staging Area - a group of changes prepared for next commit
  • Commit - a snapshot of work tree with metadata and referencing ancestors or an operation og creaticn such snapshot
  • Reference - pointer to a commit
  • Branch - pointer to last commit in the work stream
  • Tag - optionally signed pointer to a commit
  • HEAD - pointer to the tip of current branch
  • Remote - reference to another repository
  • Pull - operation of getting changes from another repository
  • Push - operation of putting changes to another repository

Config

Get and set repository or global options

Scopes: system, global and local

Clone

Clone a repository into a new directory

Init

Create an empty Git repository or reinitialize an existing one

  • --bare - create bare repository - without working directory

Add

Add file contents to the index

  • --patch - select changes to add

Rm

Remove files from the working tree and from the index

Mv

Move or rename a file, a directory, or a symlink

Commit

Record changes to the repository

  • -m - set commit message inline
  • --amend - replace last commit with new commit

Push

Update remote refs along with associated objects

  • --force - rewrite remote history

Pull

Fetch from and integrate with another repository or a local branch

  • --ff-only - abort when not fast forward
  • --rebase - rebase current branch onto incoming changes

Fetch

Download objects and refs from another repository

Merge

Join two or more development histories together

  • --ff-only - perform merge only if it is fast forward

Rebase

Reapply commits on top of another base tip

Reset

Reset current HEAD to the specified state

Remote

Manage set of tracked repositories

Branch

List, create, or delete branches

Checkout

Switch branches or restore working tree files

Log

Show commit logs

  • --oneline - single line commit messages
  • --graph - Draw history graph
  • --decorate - Show refs on commits

Status

Show the working tree status

Diff

Show changes between commits, commit and working tree, etc

Object

A chunk of data stored by git identified by hash

  • Blob - file content snapshoot
  • Tree - a file structure snapshoot referencing blobs
  • Commit - commit metadata referencing a tree
  • Tag - A reference to commit
